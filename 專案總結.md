## 六、業務流程

### 銷貨流程
```
開立銷貨單 (SalesOrder)
  → 自動建立應收帳款 (receivables)
  → 自動扣減庫存 (inventory)
  → 可選：設定收款提醒 (localStorage)
  → 列印銷貨單

客戶收款 (ReceivePayment / Receivablereport 快速付款 / SearchSales 快速付款)
  → 寫入收款紀錄 (payment_received)
  → 更新應收帳款 (receivables: paid_amount, unpaid_amount, status)
  → 更新銷貨單狀態 (sales_orders: payment_status)

修改銷貨單
  → 回補舊庫存 → 刪除舊明細 → 插入新明細 → 扣減新庫存
  → 更新應收帳款（保留已收款金額）

刪除銷貨單
  → 回補庫存 → 刪除明細 → 刪除應收帳款 → 刪除主檔
```

### 進貨流程
```
開立進貨單 (PurchaseOrder)
  → 自動建立應付帳款 (payables)
  → 自動增加庫存 (inventory)

供應商付款 (MakePayment / PayableReport 快速付款)
  → 寫入付款紀錄 (payment_made)
  → 更新應付帳款 (payables: paid_amount, unpaid_amount, status)
  → 更新進貨單狀態 (purchase_orders: payment_status)
```

## 九、頁面間通訊機制

### iframe → 父頁面（index.html）
```javascript
// 方法 1: 直接呼叫
window.parent.updateReminderBadge();

// 方法 2: postMessage
window.parent.postMessage('updateReminder', '*');
```

### 父頁面監聽
```javascript
// setInterval 輪詢（2秒）
setInterval(updateReminderBadge, 2000);

// storage 事件（跨分頁同步）
window.addEventListener('storage', updateReminderBadge);

// message 事件
window.addEventListener('message', function(e){
  if(e.data === 'updateReminder') updateReminderBadge();
});
```

## 十、共用 API 函數模式

每個頁面都有獨立的 Supabase 連線，模式如下：

```javascript
const SUPABASE_URL = 'https://xuykenpvonfqwsgbuvvj.supabase.co';
const SUPABASE_KEY = '...';
const H = {
  'apikey': SUPABASE_KEY,
  'Authorization': 'Bearer ' + SUPABASE_KEY,
  'Content-Type': 'application/json',
  'Prefer': 'return=minimal'
};

// 讀取
async function api(table, params='') {
  const r = await fetch(`${SUPABASE_URL}/rest/v1/${table}?${params}`, {headers: H});
  if (!r.ok) throw new Error(await r.text());
  return r.json();
}

// 寫入（部分頁面有）
async function apiWrite(url, method, body) {
  const opts = {method, headers: H};
  if (method !== 'DELETE') opts.body = JSON.stringify(body);
  const r = await fetch(url, opts);
  if (!r.ok) throw new Error(await r.text());
  return r;
}
```

## 十一、已知限制與待改善項目

| 項目 | 說明 | 優先級 |
